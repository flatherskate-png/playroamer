# Kate v4→v16 Port Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring the modular Roamer frontend up to parity with Kate's v16 single-file HTML by replaying her changes in order, one session at a time.

**Architecture:** Each task is a self-contained session. Read Kate's prompt or spec for the step, adapt it to reference the modular files, present the adapted prompt to Josh for review, apply after approval, then spot-check against Kate's ground-truth HTML. Sessions are sequential — complete them in order.

**Tech Stack:** Vanilla JS (no framework), CSS, HTML — split across `roamer-engine.js`, `roamer-globe.js`, `roamer-lightbox.js`, `roamer.css`, `index.html`. FastAPI backend at `localhost:8000` serves routes via API (do not replace with a static array).
